API Optimization

Rate limiting, circuit breaking, retries, idempotency, and cost tracking

GET /api/rate-limit/test Acquire a rate limit slot ▶

Sliding-window rate limiter. Allows 10 requests per 60 seconds per client. Hit this endpoint repeatedly to see rate limiting in action.

Query Parameters

NameTypeDescription
client_id string Client identifier for rate limiting. Default: user-1
Example Request
curl http://localhost:8000/api/rate-limit/test?client_id=user-1
Response — 200 OK
{
  "status": "allowed",
  "remaining": 9,
  "client_id": "user-1"
}
Response — 429 Too Many Requests
{
  "error": "Rate limit exceeded",
  "remaining": 0,
  "window_seconds": 60
}

            GET
            /api/circuit/status
            Circuit breaker state
            ▶
          

          

            
Returns the current circuit breaker state: closed (normal), open (blocking), or half_open (testing recovery).

            

              
Response

              
{
  "name": "external-api",
  "state": "closed",
  "failure_threshold": 3,
  "recovery_timeout": 30
}

            

            
            

          

        


        

          

            POST
            /api/circuit/call
            Simulate call through circuit
            ▶
          

          

            
Simulate success or failure. Send {"success": false} three times to trip the circuit open.

            

              
Request Body

              
{ "success": false }

            

            
            
            

          

        


        

          

            POST
            /api/circuit/reset
            Force reset circuit to closed
            ▶
          

          

            
Manually resets the circuit breaker back to the closed state.

            
            

          

        


        
        

          

            POST
            /api/retry/demo
            Retry with exponential backoff
            ▶
          

          

            
Demonstrates automatic retry with exponential backoff and jitter. The function fails fail_times before succeeding.

            

              
Query Parameters

              

                
NameTypeDescription

                

                  fail_times
                  integer
                  Number of times to fail before succeeding. Default: 2. Max: 3
                

              

            

            
            

          

        


        
        

          

            POST
            /api/idempotent/order
            Create order with idempotency
            ▶
          

          

            
Creates an order with idempotency protection. Send the same Idempotency-Key header twice — the second call returns the cached response.

            

              
Headers

              

                
NameRequiredDescription

                

                  Idempotency-Key
                  REQUIRED
                  Unique key to ensure the request is processed only once
                

              

            

            

              
Request Body

              
{
  "item": "Widget",
  "quantity": 3,
  "price": 12.99
}

Media Handling

            
Image transforms, MIME validation, and intelligent CDN routing

          

        


        

          

            POST
            /media/transform
            Build CDN transform URL
            ▶
          

          

            
Generates an optimized CDN transformation URL with resize, format conversion, and quality parameters.

            

              
Request Body

              
{
  "source_url": "https://cdn.example.com/photos/hero.jpg",
  "width": 800,
  "height": 600,
  "format": "webp",
  "quality": 80,
  "fit": "cover"
}

            

            
            

          

        


        

          

            GET
            /media/validate
            Validate MIME type
            ▶
          

          

            
Checks whether a content type is in the allowed media types list (images, video, audio, PDF).

            
            
            

          

        


        

          

            GET
            /media/cdn/resolve
            Route to best CDN
            ▶
          

          

            
Resolves the optimal CDN URL based on the user's region and endpoint weights.

            
            
            

          

        


        

          

            GET
            /media/cdn/all
            All CDN endpoints
            ▶
          

          

            
Returns URLs from all configured CDN endpoints for an asset.

Payment Processing

            
Webhooks, subscription state machine, and budget enforcement

          

        


        

          

            POST
            /pay/webhook/test
            Process webhook (sandbox)
            ▶
          

          

            
Send a test webhook event. The /test provider skips signature verification. Events are auto-deduplicated by ID.

            

              
Request Body

              
{
  "id": "evt_123",
  "type": "invoice.paid",
  "created": 1700000000,
  "data": { "amount": 5000 }
}

            

            
            

          

        


        

          

            POST
            /pay/subscription/create
            Create subscription
            ▶
          

          

            
Creates a new subscription starting in TRIAL state. Use the transition endpoint to move through the state machine.

            

              
Request Body

              
{
  "subscription_id": "sub_001",
  "customer_id": "cust_123",
  "plan_id": "plan_pro"
}

            

            
            

          

        


        

          

            POST
            /pay/budget/set-limit
            Set spending limit
            ▶
          

          

            
Set a monthly spending limit for a customer. Charges exceeding the limit will be blocked with a 402 response.

            

              
Request Body

              
{
  "customer_id": "cust_123",
  "amount": 100.0,
  "period": "monthly"
}

            

            
            

          

        


        

          

            POST
            /pay/budget/charge
            Record charge
            ▶
          

          

            
Records a charge against a customer's budget. Returns 402 if the charge would exceed the spending limit.

            

              
Request Body

              
{
  "customer_id": "cust_123",
  "amount": 25.0,
  "description": "API usage",
  "provider": "stripe"
}

            

            
            

          

        


        

          

            GET
            /pay/budget/status
            Check budget status
            ▶
          

          

            
Returns current spending vs. limit for a customer.
